# OpenXML WebAssembly codec

This directory contains the source-built, clean-room Office file codec. It uses the public Microsoft Open XML SDK and a public protobuf contract from `proto/open_office/artifact/v1/office_artifact.proto`.

The boundary is intentionally narrow:

- JavaScript owns the public artifact model, editing APIs, formulas, inspect/render workflows, and QA orchestration.
- `OpenOffice.OpenXmlCodec` owns bounded OOXML package import/export and structured codec diagnostics.
- `OpenOffice.OpenXmlWasm` exposes one byte-in/byte-out `[JSExport]` entrypoint. Protobuf is the only cross-language contract.
- `runtime/openxml-wasm` is reproducibly generated by `npm run build:openxml-wasm` and bundled in the npm package, so consumers do not need a local .NET SDK or Microsoft Office.

The implemented vertical slices support loss-aware XLSX, DOCX, and PPTX import/export. XLSX models workbook date systems, worksheets, primitive/cached formula cells, merged ranges, dimensions, gridline state, and frozen panes. DOCX models ordered paragraphs, limited direct run formatting, and table text. PPTX models slide order/size, top-level rectangle/ellipse shapes, and basic multi-paragraph/run text with level/alignment, bold/italic, font size/family, and solid RGB color. Source-bound text edits require unchanged paragraph/run topology and preserve unmodeled native properties behind a residual hash. Every imported DOCX body block and PPTX slide/shape-tree element carries source position, XML digest, semantic digest, and editability evidence: unchanged advanced WordprocessingML or PresentationML remains in the validated source package, while attempted edits to not-yet-modeled structures fail closed.

All three formats retain a budget-checked source-package snapshot bound to SHA-256 identity and the observed opaque part/relationship graph. A second export updates only safely modeled content through Open XML SDK, validates the complete package against Office 2021, then rejects the result unless every unmodeled part and relationship is still present with the same digest/inventory. PPTX additionally compares every package part byte-for-byte except the explicitly changed slide XML. Existing JavaScript codecs remain the explicit authoring fallback for features not yet represented by the public wire schema.

Run the native tests with:

```bash
dotnet test native/OpenXmlWasm/OpenXmlWasm.sln --configuration Release
```

Rebuild the bundled runtime with:

```bash
dotnet workload install wasm-tools wasm-experimental --skip-manifest-update
npm run build:openxml-wasm
```
