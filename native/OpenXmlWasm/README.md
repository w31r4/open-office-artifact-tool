# OpenXML WebAssembly codec

This directory contains the source-built, clean-room Office file codec. It uses the public Microsoft Open XML SDK and a public protobuf contract from `proto/open_office/artifact/v1/office_artifact.proto`.

The boundary is intentionally narrow:

- JavaScript owns the public artifact model, editing APIs, formulas, inspect/render workflows, and QA orchestration.
- `OpenOffice.OpenXmlCodec` owns bounded OOXML package import/export and structured codec diagnostics.
- `OpenOffice.OpenXmlWasm` exposes one byte-in/byte-out `[JSExport]` entrypoint. Protobuf is the only cross-language contract.
- `runtime/openxml-wasm` is reproducibly generated by `npm run build:openxml-wasm` and bundled in the npm package, so consumers do not need a local .NET SDK or Microsoft Office.

The implemented vertical slices support loss-aware XLSX, DOCX, and PPTX import/export. XLSX models workbook date systems, worksheets, primitive/cached formula cells, merged ranges, dimensions, gridline state, and frozen panes. DOCX models ordered paragraphs, limited direct run formatting, and table text. PPTX models slide order/size, top-level rectangle/ellipse shapes, and bounded paragraphs with ordered text, field, and line-break inlines; paragraph tab stops, direct left margins, hanging indents, point/multiplier line/before/after spacing, and `a:defRPr` bold/italic/size/Latin-font/solid-RGB-or-theme styles; shape-local `a:lstStyle` levels 0 through 8 through the same paragraph-property codecs; `a:bodyPr` insets, vertical anchoring, wrapping, and AutoFit mode; level/alignment; character/auto-number/no-bullet markers; content-addressed PNG/JPEG/GIF/safe-SVG or non-fetched external picture bullets; marker font/solid-RGB-or-theme color/fixed-or-percentage size plus follow-text choices; inline bold/italic, font size/family, solid RGB color; and click links for external absolute URIs, internal slide jumps, or the five standard relative slide-show actions. `PptxParagraphPropertiesCodec` coordinates the shared `TextParagraphPropertiesType` subset, `PptxListStyleCodec` alone owns `lvl1pPr` through `lvl9pPr` topology, and `PptxBodyPropertiesCodec` owns the bounded text-frame attributes and AutoFit child choice. A shared asset catalog owns durable picture bytes while the slide-scoped context owns relationship IDs and safely tracked media additions. Source-bound text edits require unchanged paragraph and inline-kind topology. They preserve transformed or unresolved picture bullets, transformed/non-theme marker colors, custom-show/mouse-over links, field-local paragraph properties, break run properties, underline/script-font/default-run transforms, unknown list-level attributes/children, attributed normal-AutoFit parameters, text rotation/vertical writing/columns/overflow/WordArt/3D, and other unmodeled native properties behind residual hashes; direct margins/indents, spacing slots, modeled default-run properties, modeled list levels, and bounded body properties can be replaced or explicitly removed. Replacing an unknown marker, marker style, default-run color, AutoFit choice, click action, or inline kind fails closed. Every imported DOCX body block and PPTX slide/shape-tree element carries source position, XML digest, semantic digest, and editability evidence: unchanged advanced WordprocessingML or PresentationML remains in the validated source package, while attempted edits to not-yet-modeled structures fail closed.

All three formats retain a budget-checked source-package snapshot bound to SHA-256 identity and the observed opaque part/relationship graph. A second export updates only safely modeled content through Open XML SDK, validates the complete package against Office 2021, then rejects the result unless every unmodeled part and relationship is still present with the same digest/inventory. PPTX additionally compares every package part byte-for-byte except an explicitly changed slide XML, its relationship part for a tracked modeled relationship, `[Content_Types].xml` for a new media type, and the exact tracked image part. Existing JavaScript codecs remain the explicit authoring fallback for features not yet represented by the public wire schema.

Run the native tests with:

```bash
dotnet test native/OpenXmlWasm/OpenXmlWasm.sln --configuration Release
```

Rebuild the bundled runtime with:

```bash
dotnet workload install wasm-tools wasm-experimental --skip-manifest-update
npm run build:openxml-wasm
```
