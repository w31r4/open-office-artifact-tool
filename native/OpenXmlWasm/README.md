# OpenXML WebAssembly codec

This directory contains the source-built, clean-room Office file codec. It uses the public Microsoft Open XML SDK and a public protobuf contract from `proto/open_office/artifact/v1/office_artifact.proto`.

The boundary is intentionally narrow:

- JavaScript owns the public artifact model, editing APIs, formulas, inspect/render workflows, and QA orchestration.
- `OpenOffice.OpenXmlCodec` owns bounded OOXML package import/export and structured codec diagnostics.
- `OpenOffice.OpenXmlWasm` exposes one byte-in/byte-out `[JSExport]` entrypoint. Protobuf is the only cross-language contract.
- `runtime/openxml-wasm` is reproducibly generated by `npm run build:openxml-wasm` and bundled in the npm package, so consumers do not need a local .NET SDK or Microsoft Office.

The implemented vertical slices support loss-aware XLSX, DOCX, and PPTX import/export. XLSX models workbook date systems, worksheets, primitive/cached formula cells, merged ranges, dimensions, gridline state, and frozen panes. DOCX models ordered paragraphs, limited direct run formatting, and table text. PPTX models slide order/size, the hash-bound Slide → Layout → Master identity chain, bounded master `p:txStyles` title/body/other levels, top-level rectangle/ellipse shapes, and bounded paragraphs with ordered text, field, and line-break inlines; paragraph tab stops, direct left margins, hanging indents, point/multiplier line/before/after spacing, and `a:defRPr` bold/italic/size/Latin-font/solid-RGB-or-theme styles; shape-local `a:lstStyle` levels 0 through 8 through the same paragraph-property codecs; `a:bodyPr` insets, vertical anchoring, wrapping, AutoFit, bounded rotation, horizontal/vertical/vertical270 text, horizontal/vertical overflow, one through sixteen columns with spacing/RTL flow, and upright text; level/alignment; character/auto-number/no-bullet markers; content-addressed PNG/JPEG/GIF/safe-SVG or non-fetched external picture bullets; marker font/solid-RGB-or-theme color/fixed-or-percentage size plus follow-text choices; inline bold/italic, font size/family, solid RGB color; and click links for external absolute URIs, internal slide jumps, or the five standard relative slide-show actions. `PptxParagraphPropertiesCodec` coordinates the shared `TextParagraphPropertiesType` subset, `PptxListStyleCodec` owns shape-local level topology, `PptxMasterTextStylesCodec` owns only the modeled level defaults inside one master's `p:txStyles`, and `PptxBodyPropertiesCodec` owns bounded text-frame attributes and AutoFit. A shared asset catalog owns durable picture bytes while `PptxPartContext` keeps every image/link relationship local to its Slide, Master, or Layout owner. Source-bound text edits require unchanged paragraph and inline-kind topology; master/layout topology and slide layout bindings are currently read-only. Modeled master-level deletion scrubs only owned properties and removes the native level only when no residual attributes or children remain. Transformed or unresolved picture bullets, transformed/non-theme marker colors, custom-show/mouse-over links, field-local paragraph properties, break run properties, underline/script-font/default-run transforms, unknown list-level attributes/children, attributed normal-AutoFit parameters, WordArt vertical modes/warp/3D, anchor-centering and compatibility flags, and other unmodeled native properties stay behind residual hashes. Replacing an unknown marker, marker style, default-run color, AutoFit choice, click action, inline kind, layout, or graph owner fails closed. Every imported DOCX body block and PPTX master/layout/slide/shape-tree element carries source position or part locator, XML digest, semantic digest where editable, and editability evidence.

All three formats retain a budget-checked source-package snapshot bound to SHA-256 identity and the observed opaque part/relationship graph. A second export updates only safely modeled content through Open XML SDK, validates the complete package against Office 2021, then rejects the result unless every unmodeled part and relationship is still present with the same digest/inventory. PPTX additionally compares every package part byte-for-byte except an explicitly changed slide or master XML, its relationship part for a tracked modeled relationship, `[Content_Types].xml` for a new media type, and the exact tracked image part. Existing JavaScript codecs remain the explicit authoring fallback for features not yet represented by the public wire schema.

Run the native tests with:

```bash
dotnet test native/OpenXmlWasm/OpenXmlWasm.sln --configuration Release
```

Rebuild the bundled runtime with:

```bash
dotnet workload install wasm-tools wasm-experimental --skip-manifest-update
npm run build:openxml-wasm
```
