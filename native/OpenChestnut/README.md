# OpenChestnut

OpenChestnut is the source-built, clean-room Office package codec in this repository. It uses the public Microsoft Open XML SDK and this project's independent protobuf contract from `proto/open_office/artifact/v1/office_artifact.proto`.

The boundary is intentionally narrow:

- JavaScript owns the public artifact model, editing APIs, formulas, inspect/render workflows, and QA orchestration.
- `OpenChestnut.Codec` owns bounded OOXML package import/export and structured codec diagnostics.
- `OpenChestnut.Runtime` exposes one byte-in/byte-out `[JSExport]` entrypoint. Protobuf is the only cross-language contract.
- `runtime/open-chestnut` is reproducibly generated by `npm run build:open-chestnut` and bundled in the npm package, so consumers do not need a local .NET SDK or Microsoft Office.

The historical `codecs/openxml-wasm` JavaScript subpath and matching npm scripts are deprecated compatibility shims. They delegate one-way to OpenChestnut; no legacy C# project, assembly, runtime bundle, or second implementation remains.

The implemented vertical slices support loss-aware XLSX, DOCX, and PPTX import/export. XLSX models workbook date systems, worksheets, primitive/cached formula cells, merged ranges, dimensions, gridline state, and frozen panes. DOCX models ordered paragraphs, limited direct run formatting, table text, bounded whole-paragraph external/internal hyperlinks, bounded whole-paragraph `w:fldSimple` fields, and direct-numbered single-run paragraphs. `DocxHyperlinkCodec` edits target, text, tooltip, history, and owned relationships. `DocxFieldCodec` authors or edits one-result-run simple fields for a fixed non-fetching command catalog. `DocxTableCodec` edits text only in fixed row/cell topology whose cells contain exactly one paragraph/run/text node; residual hashes protect table, row, cell, paragraph, and run formatting, while merged, nested, multi-paragraph, and multi-run cells remain source-preserved and read-only. `DocxNumberedParagraphCodec` resolves direct `w:numPr` through a read-only numbering-part stream and permits text-only edits while residual-protecting the paragraph assignment and shared definition graph. PPTX models slide order/size, the hash-bound Slide → Layout → Master identity chain, bounded master `p:txStyles` title/body/other levels, direct Master/Layout backgrounds, owner-local Master/Layout placeholder text, top-level rectangle/ellipse shapes, and bounded paragraphs with ordered text, field, and line-break inlines; paragraph tab stops, direct left margins, hanging indents, point/multiplier line/before/after spacing, and `a:defRPr` bold/italic/size/Latin-font/solid-RGB-or-theme styles; shape-local `a:lstStyle` levels 0 through 8 through the same paragraph-property codecs; `a:bodyPr` insets, vertical anchoring, wrapping, AutoFit, bounded rotation, horizontal/vertical/vertical270 text, horizontal/vertical overflow, one through sixteen columns with spacing/RTL flow, and upright text; level/alignment; character/auto-number/no-bullet markers; content-addressed PNG/JPEG/GIF/safe-SVG or non-fetched external picture bullets; marker font/solid-RGB-or-theme color/fixed-or-percentage size plus follow-text choices; inline bold/italic, font size/family, solid RGB color; and click links for external absolute URIs, internal slide jumps, or the five standard relative slide-show actions. `PptxParagraphPropertiesCodec` coordinates the shared `TextParagraphPropertiesType` subset, `PptxListStyleCodec` owns shape-local level topology, `PptxMasterTextStylesCodec` owns only the modeled level defaults inside one master's `p:txStyles`, `PptxPlaceholderCodec` owns direct `p:ph` identity plus local `a:txBody`, and `PptxBodyPropertiesCodec` owns bounded text-frame attributes and AutoFit. A shared asset catalog owns durable picture bytes while `PptxPartContext` keeps every image/link relationship local to its Slide, Master, or Layout owner. Source-bound text edits require unchanged paragraph and inline-kind topology; master/layout/placeholder topology, placeholder identity and shape-level properties, and slide layout bindings are read-only. Modeled master-level deletion scrubs only owned properties and removes the native level only when no residual attributes or children remain. Transformed or unresolved picture bullets, transformed/non-theme marker colors, custom-show/mouse-over links, field-local paragraph properties, break run properties, underline/script-font/default-run transforms, unknown list-level attributes/children, attributed normal-AutoFit parameters, WordArt vertical modes/warp/3D, anchor-centering and compatibility flags, and other unmodeled native properties stay behind residual hashes. Replacing an unknown marker, marker style, default-run color, AutoFit choice, click action, inline kind, layout, placeholder owner/identity, or graph owner fails closed. Every imported DOCX body block and PPTX master/layout/slide/shape-tree element carries source position or part locator, XML digest, semantic digest where editable, and editability evidence.

`PptxBackgroundCodec` independently owns direct Master/Layout `p:bgPr/a:solidFill` and `p:bgRef` choices with RGB or one of the 16 DrawingML theme tokens. It never serializes a computed inherited color. Gradient, pattern, image, group, transformed, effect-bearing, and explicit-background removal semantics remain source-bound and fail closed.

`PptxPlaceholderCodec` independently enumerates direct placeholder shapes by owner-local shape-tree position, normalizes native default `type="obj"` and `idx="0"`, and binds name/type/index/text semantics to the original XML digest. It edits only supported local `a:txBody` semantics; position, geometry, fill, shape style, other `p:ph` attributes, and inherited/effective content remain residual-hash protected.

All three formats retain a budget-checked source-package snapshot bound to SHA-256 identity and the observed opaque part/relationship graph. A second export updates only safely modeled content through Open XML SDK, validates the complete package against Office 2021, then rejects the result unless every unmodeled part and relationship is still present with the same digest/inventory. PPTX additionally compares every package part byte-for-byte except an explicitly changed slide, master, or layout XML, its relationship part for a tracked modeled relationship, `[Content_Types].xml` for a new media type, and the exact tracked image part. Existing JavaScript codecs remain the explicit authoring fallback for features not yet represented by the public wire schema.

Run the native tests with:

```bash
dotnet test native/OpenChestnut/OpenChestnut.sln --configuration Release
```

Rebuild the bundled runtime with:

```bash
dotnet workload install wasm-tools wasm-experimental --skip-manifest-update
npm run build:open-chestnut
```
