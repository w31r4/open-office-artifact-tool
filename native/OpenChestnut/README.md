# OpenChestnut

OpenChestnut is the source-built, clean-room Office package codec in this repository. It uses the public Microsoft Open XML SDK and this project's independent protobuf contract from `proto/open_office/artifact/v1/office_artifact.proto`.

For PPTX, `PptxNativeObjectCatalog` performs namespace-aware classification of source-bound `contentPart`, OLE, SmartArt/diagram, picture, group, connector, and generic graphic-frame elements. It records relationship-valued XML attributes and a cycle-safe, budgeted transitive closure of internal part paths while leaving canonical content types, relationships, hashes, and source bytes in `ArtifactEnvelope.opaque_opc`. Recognized top-level OLE/diagram `p:graphicFrame` elements and `contentPart`-carrying `p:grpSp` elements become placement-editable only when their complete native outer transform and non-visual name are present. Export may replace that name and outer frame (plus the OLE preview transform that represents the same frame), while raw XML residuals, group child coordinates, relationship references, content types, part bytes, and graph topology remain hash-bound. Generic/nested/incomplete profiles stay read-only. Missing IDs/parts, duplicate relationship IDs, unsafe targets, traversal-budget excess, malformed frames, wire editability escalation, and graph tampering return structured diagnostics before or after mutation.

`PptxOleWorkbookCodec` adds one separately hash-bound payload exception. Import exposes `PresentationOleWorkbook` only when a recognized top-level OLE has exactly one internal `/package` relationship to one non-shared part with the standard XLSX content type. Export accepts a content-addressed `asset/presentation/ole-workbook/<sha256>` replacement, applies the ordinary OPC budgets, requires a workbook with at least one worksheet, validates it with Open XML SDK for Office 2021, writes only the existing `EmbeddedPackagePart`, and proves the output digest without changing its path, content type, relationship, preview, or surrounding graph. It does not create OLE objects, regenerate previews, accept non-XLSX payloads, or expose arbitrary package-part mutation.

The boundary is intentionally narrow:

- JavaScript owns the public artifact model, editing APIs, formulas, inspect/render workflows, and QA orchestration.
- `OpenChestnut.Codec` owns bounded OOXML package import/export and structured codec diagnostics.
- `OpenChestnut.Runtime` exposes one byte-in/byte-out `[JSExport]` entrypoint. Protobuf is the only cross-language contract.
- `runtime/open-chestnut` is reproducibly generated by `npm run build:open-chestnut` and bundled in the npm package, so consumers do not need a local .NET SDK or Microsoft Office.

The XLSX slice uses shared `XlsxSortStateCodec` parsing, validation, writing, patching, selector handling, and semantic hashing across worksheet, Table, and QueryTable contexts. `WorksheetArtifact.sort_state` additive field 10 authors or edits worksheet-level sorting, while presence-aware `SpreadsheetTableSortStateArtifact.column_sort` field 5 distinguishes omission, explicit row sorting, and left-to-right column sorting. Ordinary sorts use unique single-column conditions with the containing row span; `columnSort=true` uses unique single-row conditions with the containing column span. SpreadsheetML permits `columnSort` only outside AutoFilter ranges, so source-free worksheet and source-bound QueryTable refresh sorts support it while Table AutoFilter sorts reject either explicit boolean value. Unsupported geometry stays hidden in the hash-bound source and cannot be replaced. `XlsxTableCodec` owns worksheet `tableParts` locators and TableDefinitionPart relationships, authors bounded source-free tables, and edits name/range, header/totals/filter, rich column, style-name, and first/last/row/column-stripe semantics while keeping imported part paths and relationship IDs fixed. Rich columns cover calculated-column formulas, array flags, built-in or custom totals functions, labels, and totals formulas; bounded AutoFilters cover exact values/blanks, calendar-aware grouped dates, one/two comparison criteria, standard dynamic types with optional thresholds, top/bottom item-or-percent criteria, standard SpreadsheetML icon sets, and stable cell-fill/font-color selectors keyed by zero-based table column index. Exact values and grouped dates are alternatives within one native `<filters>` branch. Sort state also carries case sensitivity, optional `none`/`pinYin`/`stroke` method, ordered custom-list/value/icon/color selectors, and optional icon IDs; the public wire carries stable colors, not `dxfId`. `XlsxCellStyleCodec` recognizes exact fill-only/font-color-only differential formats, deduplicates source-free records, and appends source-bound replacements without rewriting shared source entries. Cell values/formulas remain owned by `XlsxCodec`. `XlsxQueryTableCodec` recognizes one existing TableDefinitionPart → QueryTablePart relationship, requires its `connectionId` to identify an entry in the source ConnectionsPart, and exposes presence-aware root policy plus one bounded `queryTableRefresh` profile. Recognized refresh metadata covers sort/filter-layout preservation, field-ID wrapping, last-refresh headers, version/next-ID/unbound-column counters, field name/data-bound/row-number/formula-fill/clipped flags, ordered deleted-field names, and refresh-local row/column sort state. Source-bound edits may change those values or select another connection already present in the same hash-bound ConnectionsPart, but field IDs/order/count/table-column bindings, deleted-field count/order, and sort presence/condition count remain immutable. Shared workbook-level `XlsxConnectionCodec` owns the ConnectionsPart exactly once. Its additive wire projection exposes only database/type-5 root name, optional description, keep-alive, interval, background, refresh-on-load, and save-data semantics plus hash-bound source identity. Connection count/order and ID/type/refreshed-version remain immutable. Provider strings, commands, credentials, source/ODC paths, child graphs, unknown root attributes, extensions, and unsupported connection types stay hidden and exact-preserved. `XlsxQuerySortStateCodec` patches the shared bounded sort vocabulary in place and preserves sort extensions. Formula-fill fields must be explicitly unbound; clipped fields must be explicitly data-bound. The codec never creates, deletes, reorders, changes the type of, or executes an external connection. Opaque deleted-field/sort branches stay independently hidden while safe root/field edits continue; an opaque complete refresh hides the refresh, and unsupported companion relationships keep the complete table hidden/read-only. Multi-property differential styles and other complex table XML remain byte-exact and read-only; imported table addition/removal, QueryTable graph topology changes, and field/deleted-field/sort topology changes fail closed.

`XlsxDefinedNameCodec` owns the bounded `workbook.xml/definedNames` projection. Additive `WorkbookArtifact.defined_names` field 6 carries ordinary sheet-qualified A1 range names with optional worksheet scope, comment, hidden state, and a hash-bound ordinal locator. Source-free export authors that profile. Imported recognized entries may edit their bounded semantics in place, but count/order/source identity are immutable. Reserved `_xl*` names, constants/formulas, macro/function attributes, extensions, and malformed profiles stay hidden and exact-preserved; collision, topology change, or binding tamper fails closed.

`XlsxCalculationCodec` owns the bounded `workbook.xml/calcPr` projection behind additive `WorkbookArtifact.calculation` field 7. It authors or hash-binds automatic/automatic-except-tables/manual mode, presence-aware save/full-recalculation/full-precision flags, and bounded iterative count/delta settings. Imported element presence is immutable, application `calcId` remains residual, and R1C1, completed-state, concurrent, extended, or invalid profiles stay opaque and exact-preserved. The codec records host calculation policy; it does not execute formula recalculation.

`XlsxWorksheetMetadataCodec` owns workbook `<sheet>` name and visibility projection. Additive `WorksheetArtifact.visibility` field 11 uses this project's visible/hidden/very-hidden enum, while field 12 binds ordinal, relationship ID, native sheet ID, element hash, semantic hash, and editability. Source-bound edits retain sheet count/order/identity. Unknown state tokens remain exact-preserved and read-only, and every export requires at least one recognized visible worksheet.

`XlsxChartCodec` owns the bounded worksheet GraphicFrame → ChartPart projection behind additive `WorksheetArtifact.charts` field 14. The public wire models one bar, line, or pie plot with name/title/legend, shared string categories, ordered numeric series, optional category/value formulas, deterministic caches, and one native-EMU one-cell/two-cell/absolute anchor. `XlsxChartSeriesStyleCodec` owns optional series field 5, accepting only a direct `c:ser/c:spPr/a:solidFill/a:srgbClr` as a six-digit RGB value; it can author, add, remove, or change that exact node without replacing other shape properties. Theme, gradient, no-fill, transformed color, duplicate, or otherwise unrecognized fill graphs make the chart read-only and exact-preserved. `XlsxChartSeriesLineStyleCodec` owns additive series field 6 as one direct `a:ln`: optional six-digit RGB color, one of five preset dash styles, and optional 0–1,584 point width. It can add, remove, or change that exact line while theme/transformed colors, compound/cap/join/arrow/custom-dash graphs make the chart read-only and exact-preserved. `XlsxChartSeriesMarkerCodec` owns additive series field 7 as a line-only direct `c:marker`: one of ten symbols plus optional integer size 2–72. It can add, remove, or change that exact marker; marker shape properties, picture markers, extensions, duplicates, and unknown children make the whole chart read-only and exact-preserved. `XlsxChartLineOptionsCodec` owns additive chart field 15 as the required line-plot `c:grouping` scalar plus optional `c:smooth` boolean and `c:varyColors` true flag. Grouping accepts `standard`, `stacked`, or `percentStacked`; absent public grouping authors `standard`, while protobuf presence retains an explicit value. Smooth retains explicit true/false/absence. Vary-colors true authors `val="1"`; false or absence removes the optional node, while native false imports as editable false. The codec can add, remove, or change owned semantics; missing/unknown grouping values, extra attributes/children, duplicates, malformed smooth/vary-colors values, and non-line use make the whole chart read-only or fail closed. `XlsxChartTextStyleCodec` owns chart field 14 `title_text_style` and axis field 7 `text_style`, each limited to an explicit 1–4,000 point font size. It patches only an exact single-run title `a:rPr@sz` or canonical axis `c:txPr/a:defRPr@sz`; rich title runs and complex/conflicting `txPr` remain byte-exact but make the chart read-only. Axis `text_style` applies to tick labels, not the axis title. `XlsxChartAxisCodec` separately owns additive chart fields 12/13: primary text-category/value-axis titles and number formats, category tick-label interval, and linear value minimum/maximum/major unit. It retains explicit zero through protobuf presence and rejects inverted ranges. All modules retain their unowned ChartSpace XML. The chart codec shares the worksheet DrawingPart with `XlsxDrawingCodec` through one pre-mutation drawing hash, so a source-preserving export may edit a picture and chart in the same pass without invalidating either binding. Source-free export authors relationship-backed ChartParts; recognized imports may edit name/title/legend/formulas/caches/RGB fills/direct lines/direct markers/chart-level grouping/smoothing/vary-colors/font sizes/axes while anchor kind/geometry, chart type, series count, point count, literal/reference topology, and primary-axis identity remain fixed. Formula-referenced series names and logarithmic/date/secondary/multi-axis profiles import read-only and exact-roundtrip. External data, combo/multi-plot charts, advanced marker styling, axis-title styles, data labels, add/remove, and topology changes remain opaque or fail closed.

`XlsxWorkbookViewCodec` independently owns workbook-window active sheets and selected tab groups. Existing `WorkbookArtifact.view` field 8 remains the primary window; additive `additional_views` field 9 contains only later windows. Each view carries stable active/selected worksheet IDs and a zero-based source ordinal. Source-free export coordinates every `WorkbookView.ActiveTab` with one matching `SheetView.TabSelected` per worksheet and `workbookViewId`. Recognized imports require a complete one-per-window/per-worksheet graph, bind every element/part by hash, and keep source window count/order immutable. Source-bound edits change only the owned active/selection attributes while retaining geometry, panes, zoom, first-sheet scroll position, visibility, extensions, and residual XML. An incomplete single-window graph remains readable but selection edits fail closed; an incomplete multi-window graph stays opaque, and all source-active/selected visible tabs remain protected from hiding.

The historical `codecs/openxml-wasm` JavaScript subpath and matching npm scripts are deprecated compatibility shims. They delegate one-way to OpenChestnut; no legacy C# project, assembly, runtime bundle, or second implementation remains.

`XlsxDynamicArrayCodec` is the workbook-level owner for the bounded XLDAPR dynamic-array profile. It authors one canonical `CellMetadataPart` for source-free workbooks, maps the one-based `c@cm` index during import, preserves recognized source metadata across safe formula/value/style edits, and clears only the owned cell marker when a dynamic formula is detached. `XlsxFormulaCodec` remains the owner of the anchor formula and spill range. Adding a dynamic formula to a source package without a recognized record, replacing a mixed/opaque metadata block, or exporting a blocked spill fails closed instead of rewriting unknown package semantics.

The implemented vertical slices support loss-aware XLSX, DOCX, and PPTX import/export. XLSX models workbook date systems, the twelve workbook-theme color slots, worksheets, primitive/cached formula cells, native shared-formula groups, legacy array-formula anchors, complete static cell styles, merged ranges, dimensions, gridline state, and frozen panes. `XlsxFormulaCodec` validates complete non-overlapping shared groups and array anchors, expands follower formulas with bounded relative A1 translation, rejects data-table/unknown types, emits canonical master/follower `t/si/ref` topology, and separates formula mutation from cached-value/style mutation so unrelated edits preserve the source `<f>` XML exactly. Formula edits carrying unmodeled attributes fail closed; writing one member through the JavaScript range API detaches the complete shared group to ordinary formulas. `XlsxCellStyleCodec` is the sole SpreadsheetML stylesheet owner: it creates valid minimal source-free styles and roundtrips font, pattern fill, nine border edges, alignment, protection, and number formats. Imported edits clone the selected XF and referenced resources, append/deduplicate replacements, and retain unmodeled properties; `XlsxNumberFormatCodec` remains the pure built-in-format vocabulary helper. `XlsxThemeCodec` authors a complete theme and source-binds its name plus `dk1`/`lt1`/`dk2`/`lt2`/six accents/hyperlink/followed-hyperlink RGB slots. Recognized imported themes retain font schemes, format schemes, extensions, and other residual XML during a modeled edit. System colors map through `lastClr` and stay intact when unchanged; per-color transforms stay residual unless that exact slot is explicitly replaced. HSL, preset, scheme, and other complex color choices remain exact-preserved but uneditable. Invalid style references, colors, enumerations, ranges, missing custom formats, unmapped locale-dependent built-ins, oversized codes, and control characters fail closed. Gradient fills, HSL/preset/scheme theme colors, theme font/format-scheme editing, and named/cell style inheritance remain source-preserved rather than modeled. DOCX models ordered paragraphs, limited direct run formatting, table text, merge geometry, a bounded direct table-formatting profile, bounded whole-paragraph external/internal hyperlinks, bounded whole-paragraph `w:fldSimple` fields, and direct or paragraph/numbering-style-linked numbered single-run paragraphs. `DocxHyperlinkCodec` edits target, text, tooltip, history, and owned relationships. `DocxFieldCodec` authors or edits one-result-run simple fields for a fixed non-fetching command catalog. `DocxTableGeometry` derives logical columns, `gridSpan`, `vMerge`, effective row spans, row offsets, and per-cell editability without changing imported topology. `DocxTableCodec` authors complete, contiguous source-free physical-cell geometry after exact span/merge-chain validation and emits a minimal default `TableGrid` style. `DocxTableFormatting` separately owns a complete fixed-layout profile: dxa table/indent/grid/cell widths, four margins, uniform RGB borders, and first-row fill/bold. Width sums, logical column counts, RGB values, and Word border-size bounds are validated before mutation; merged cell widths are derived from the logical grid. Custom style graphs, gaps, overlaps, mismatched continuations, partial/theme/conditional/per-cell formatting, nonempty continuation text, and row-span mismatches fail closed. For imported tables it edits text only in simple unmerged, horizontally merged, or vertical-origin cells with exactly one paragraph/run/text node; continuation cells are read-only, direct formatting is source-bound, and residual hashes protect all table, row, cell, paragraph, and run formatting. Invalid merge chains and nested, multi-paragraph, and multi-run tables remain source-preserved and read-only. `DocxNumberedParagraphCodec` owns the paragraph assignment and text-only edit, while `DocxNumberingResolver` follows `w:lvl/w:pStyle`, `w:styleLink`, `w:numStyleLink`, numbering-style `numId`, and instance overrides through cached read-only styles/numbering XML. Paragraph-style `ilvl` is deliberately ignored; direct paragraph `ilvl` remains authoritative. Numbering-style identity is additive wire metadata and remains source-bound with the rest of the graph. Missing, duplicate, cyclic, over-deep, ambiguous, or unresolved graphs remain source-preserved and read-only. PPTX models slide order/size, the hash-bound Slide → Layout → Master package identity chain, bounded master `p:txStyles` title/body/other levels, direct Master/Layout backgrounds, owner-local Master/Layout placeholder text, top-level rectangle/ellipse shapes, and bounded paragraphs with ordered text, field, and line-break inlines; paragraph tab stops, direct left margins, hanging indents, point/multiplier line/before/after spacing, and `a:defRPr` bold/italic/size/Latin-font/solid-RGB-or-theme styles; shape-local `a:lstStyle` levels 0 through 8 through the same paragraph-property codecs; `a:bodyPr` insets, vertical anchoring, wrapping, AutoFit, bounded rotation, horizontal/vertical/vertical270 text, horizontal/vertical overflow, one through sixteen columns with spacing/RTL flow, and upright text; level/alignment; character/auto-number/no-bullet markers; content-addressed PNG/JPEG/GIF/safe-SVG or non-fetched external picture bullets; marker font/solid-RGB-or-theme color/fixed-or-percentage size plus follow-text choices; inline bold/italic, font size/family, solid RGB color; and click links for external absolute URIs, internal slide jumps, or the five standard relative slide-show actions. `PptxParagraphPropertiesCodec` coordinates the shared `TextParagraphPropertiesType` subset, `PptxListStyleCodec` owns shape-local level topology, `PptxMasterTextStylesCodec` owns only the modeled level defaults inside one master's `p:txStyles`, `PptxPlaceholderCodec` owns direct `p:ph` identity plus local `a:txBody`, and `PptxBodyPropertiesCodec` owns bounded text-frame attributes and AutoFit. A shared asset catalog owns durable picture bytes while `PptxPartContext` keeps every image/link relationship local to its Slide, Master, or Layout owner. Source-bound text edits require unchanged paragraph and inline-kind topology; master/layout/placeholder topology, placeholder identity and shape-level properties, and slide layout bindings are read-only. Modeled master-level deletion scrubs only owned properties and removes the native level only when no residual attributes or children remain. Transformed or unresolved picture bullets, transformed/non-theme marker colors, custom-show/mouse-over links, field-local paragraph properties, break run properties, underline/script-font/default-run transforms, unknown list-level attributes/children, attributed normal-AutoFit parameters, WordArt vertical modes/warp/3D, anchor-centering and compatibility flags, and other unmodeled native properties stay behind residual hashes. Replacing an unknown marker, marker style, default-run color, AutoFit choice, click action, inline kind, layout, placeholder owner/identity, or graph owner fails closed. Every imported DOCX body block and PPTX master/layout/slide/shape-tree element carries source position or part locator, XML digest, semantic digest where editable, and editability evidence.

For the complete direct table-formatting profile, source-bound identity no longer makes the modeled values immutable. After exact recognition, `DocxTableFormatting` updates width, indent, grid/cell widths, margins, uniform borders, and header fill in place. The residual hash canonicalizes only those owned values; table styles, row properties, paragraph/run properties, and other unknown XML remain protected. Missing, partial, theme, conditional, per-cell, and style-effective formatting rejects replacement.

`DocxDirectNumbering` owns source-free direct text-marker numbering. It preflights all numbered blocks before package mutation, groups compatible levels by shared abstract identity, groups paragraphs by numbering instance, and emits schema-ordered `w:abstractNum`/`w:lvl`/`w:num` plus direct `w:numPr`. Conflicting instance, abstract, or level definitions, nonpositive starts, and style-linked graphs fail closed. Picture bullets remain outside this public wire slice.

`DocxNumberingEditPlanner` adds a bounded definition-edit transaction on top of the numbered-paragraph text slice. It preflights the complete source body and allows `numberFormat`, `start`, and `levelText` changes only when every editable top-level paragraph directly assigned to one `numId`/level requests the same target. The planner writes an instance-local `w:lvlOverride` without mutating the shared `w:abstractNum`; partial groups, nested or cross-part uses, style inheritance/linkage, identity changes, and incompatible existing overrides fail closed.

`PptxBackgroundCodec` independently owns direct Master/Layout `p:bgPr/a:solidFill` and `p:bgRef` choices with RGB or one of the 16 DrawingML theme tokens. It never serializes a computed inherited color. Gradient, pattern, image, group, transformed, effect-bearing, and explicit-background removal semantics remain source-bound and fail closed.

`PptxPlaceholderCodec` independently enumerates direct placeholder shapes by owner-local shape-tree position, normalizes native default `type="obj"` and `idx="0"`, and binds name/type/index/text semantics to the original XML digest. It edits only supported local `a:txBody` semantics; position, geometry, fill, shape style, other `p:ph` attributes, and inherited/effective content remain residual-hash protected.

All three formats retain a budget-checked source-package snapshot bound to SHA-256 identity and the observed opaque part/relationship graph. A second export updates only safely modeled content through Open XML SDK, validates the complete package against Office 2021, then rejects the result unless every unmodeled part and relationship is still present with the same digest/inventory. PPTX additionally compares every package part byte-for-byte except an explicitly changed slide, master, or layout XML, its relationship part for a tracked modeled relationship, `[Content_Types].xml` for a new media type, and the exact tracked image part. Existing JavaScript codecs remain the explicit authoring fallback for features not yet represented by the public wire schema.

Run the native tests with:

```bash
dotnet test native/OpenChestnut/OpenChestnut.sln --configuration Release
```

Rebuild the bundled runtime with:

```bash
dotnet workload install wasm-tools wasm-experimental --skip-manifest-update
npm run build:open-chestnut
```
