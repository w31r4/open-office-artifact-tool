# Clean-room coverage matrix

This document tracks the target behavior observed from four Office/PDF artifact workflows and the current implementation status of `open-office-artifact-tool`.

Status legend:

- `done`: implemented and covered by smoke tests.
- `partial`: API or minimal behavior exists, but fidelity/edge cases remain.
- `todo`: not implemented yet.

## Implementation priorities

1. Continue moving format-specific package and source-reference decisions behind family modules; next targets are richer Pivot calculated-field semantics, presentation modern comments, and newer DOCX comment IDs/people behavior.
2. Continue broadening Excel formula/coercion coverage on top of the shipped dependency graph.
3. Improve presentation modern-comment, chart-style, placeholder/list, and broader third-party master/theme fidelity on top of the shipped per-master native Theme graph and table/chart/image/connector parts.
4. Broaden render-backed baselines across platforms and add non-translation registration beyond the shipped Playwright/sharp/canvas/Poppler/LibreOffice/native Office adapters.
5. Improve DOCX pagination, inherited header/footer layout, and newer comment IDs/people interoperability on top of the shipped commentsExtended and native Theme/run-style codecs, real render gate, and design presets.
6. Continue robust arbitrary-PDF extraction, font/image fidelity, accessibility, and cross-platform render baselines on top of the shipped PDF.js/Poppler workflow.
7. Expand `help` catalogs from the observed skill docs into generated, testable API records.
