# Clean-room coverage matrix

This document tracks the target behavior observed from four Office/PDF artifact workflows and the current implementation status of `open-office-artifact-tool`.

Status legend:

- `done`: implemented and covered by smoke tests.
- `partial`: API or minimal behavior exists, but fidelity/edge cases remain.
- `todo`: not implemented yet.

## Implementation priorities

1. Migrate the remaining hand-written XML emitters onto the shipped safe OOXML inspect/patch layer and add relationship/content-type consistency builders.
2. Continue broadening Excel formula/coercion coverage on top of the shipped dependency graph.
3. Replace presentation table/chart/image placeholders with true native PPTX OOXML parts while preserving inspect/resolve/layout behavior.
4. Add additional raster/render adapters (`sharp`, canvas, Poppler/LibreOffice, native Office) and render-backed QA on top of the shipped Playwright adapter. (sharp, canvas, Poppler, LibreOffice, native Office adapters shipped; `visualQaArtifact` compares PNG/JPEG/WebP/PPM pixels, while PDF page-set diff orchestration remains partial.)
5. Add DOCX real render gate, design presets, and richer style fidelity.
6. Continue robust arbitrary-PDF extraction, font/image fidelity, accessibility, and cross-platform render baselines on top of the shipped PDF.js/Poppler workflow.
7. Expand `help` catalogs from the observed skill docs into generated, testable API records.
